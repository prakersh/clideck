Termix v1.1 Workplan
====================

1. Sound Notifications & Per-Session Mute
   - Notification sound on working-to-idle transitions (default: on)
   - 7 bundled sounds: Default, Soft, Bold, Echo, Musical, Bleep, Space
   - Sound picker + preview button in Settings > Notifications
   - Global toggle: notifySoundEnabled (on by default)
   - Per-session mute via context menu (mutes sound + browser notifications)
   - Mute indicator icon in sidebar row
   - Mute state persisted in sessions.json across restarts
   - Mute broadcast to all connected clients for multi-tab sync
   - .mp3 MIME type added to static server
   - Notification rules info box in settings panel
   - Sound: plays for background sessions only (not the one you're viewing)
   - Browser: only when Termix tab is not in focus
   Files changed: index.html, settings.js, terminals.js, app.js, state.js,
                  server.js, sessions.js, handlers.js, config.js
   Assets added:  public/fx/*.mp3 (7 files)

2. npm Publishing (termix-cli)
   - Package name: termix-cli
   - CLI entry point: bin/termix.js (npx termix-cli)
   - Runtime data moved to ~/.termix/ (config, sessions, transcripts, custom themes)
   - Legacy file migration on first run after upgrade
   - os.homedir() instead of process.env.HOME
   - .npmignore for clean package (no src/, config.json, sessions.json, etc.)
   - public/tailwind.css committed (was gitignored)
   - prepublishOnly builds CSS automatically
   - Published: termix-cli@1.1.0
   Files added:   bin/termix.js, paths.js, .npmignore
   Files changed: package.json, .gitignore, README.md, config.js, sessions.js,
                  transcript.js, themes.js
