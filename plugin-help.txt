Termix Plugin Guide
====================

Folder structure
----------------
  ~/.termix/plugins/<plugin-id>/
    termix-plugin.json   (manifest — required)
    index.js             (backend entry — required)
    client.js            (frontend entry — optional)
    public/              (static assets — optional, served at /plugins/<id>/*)

Manifest (termix-plugin.json)
-----------------------------
{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "1.0.0",
  "settings": [
    { "key": "mode", "label": "Mode", "type": "select", "options": ["fast", "slow"], "default": "fast" },
    { "key": "enabled", "label": "Enable feature", "type": "toggle", "default": true },
    { "key": "limit", "label": "Max items", "type": "number", "min": 1, "max": 100, "default": 10 },
    { "key": "apiKey", "label": "API Key", "type": "text", "default": "", "description": "Optional API key" }
  ]
}

Fields: id (unique), name (display), version, settings (optional array).
Setting types: text, toggle, select, number.
Select options: array of strings, or objects with { value, label }.
Number supports optional min/max.

Backend API (index.js)
----------------------
module.exports = {
  init(api) {
    // api.version         — API version (1)
    // api.pluginId        — this plugin's ID
    // api.pluginDir       — absolute path to this plugin's folder
    // api.log(msg)        — log with plugin prefix

    // Session hooks
    api.onSessionInput((sessionId, data) => {
      // Transform input before it reaches the PTY. Return a string to replace.
      return data;
    });
    api.onSessionOutput((sessionId, data) => {
      // Observe PTY output (read-only).
    });
    api.onStatusChange((sessionId, working) => {
      // Called when a session transitions between working/idle.
    });

    // Frontend messaging (namespaced as plugin.<id>.<event>)
    api.sendToFrontend('my-event', { key: 'value' });
    api.onFrontendMessage('my-action', (msg) => { /* handle */ });

    // Sessions
    api.getSessions();              // [{ id, name, cwd, commandId, themeId, projectId }]
    api.getSession(id);             // single session or null

    // Toolbar UI slot
    api.addToolbarAction({
      id: 'do-thing',
      title: 'Do Thing',
      icon: '<svg>...</svg>',       // inline SVG, 16x16
    });

    // Settings (values come from manifest defaults + user overrides)
    api.getSetting('mode');          // current value for key
    api.getSettings();               // { mode: 'fast', enabled: true, ... }
    api.onSettingsChange((key, value) => {
      // Called when the user changes a setting in the UI.
    });

    // Cleanup
    api.onShutdown(() => { /* called on server shutdown */ });
  }
};

UI
--
Installed plugins appear in the Plugins panel (puzzle-piece icon in the
sidebar rail). If a plugin declares settings in its manifest, they are
rendered as editable controls (toggles, dropdowns, text fields, number
inputs) in the panel. Changes take effect immediately.

Plugins are loaded on server startup. After adding or removing a plugin
folder, restart Termix to pick up the change.

Notes
-----
- Plugins run in-process with full Node.js access — same trust model as npm
  packages. Only install plugins you trust or have reviewed.
- Hook errors are caught and logged — one plugin can't crash others.
- Input hooks run as a chain: each hook's return replaces the data for the next.
- Frontend messages are namespaced automatically: plugin.<id>.<event>.
- Setting values are validated against the manifest (type, options, min/max).
- Static files in public/ are served at /plugins/<id>/<filename>.
